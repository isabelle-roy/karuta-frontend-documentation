À l'aide de la métadonnée Partager, il possible de permettre aux utilisateurs de partager leur portfolio, au complet ou en partie, avec un tiers et même de donner à ce tiers des droits d'édition sur les éléments partagés. L'utilisateur procède au partage en cliquant sur un bouton, ce qui déclenche l'envoi d'un courriel au tiers avec un hyperlien vers les éléments partagés.

Le partage permet de réaliser deux scénario :

  • avertir un acteur du portfolio qu’il doit intervenir ;
  • permettre à un utilisateur qui n’a pas actuellement les droits d’intervenir dans un portfolio de le faire.

Dans le portfolio

Dans l'exemple ci-dessus, lorsque l’étudiant clique sur Partager avec un pair, le système envoie un courriel au pair avec un hyperlien vers la section du portfolio dans laquelle il pourra inscrire son commentaire.

Il y a plusieurs options pour les conditions de sécurité du partage ; voici quelques cas d’utilisation usuels :

Cas 1 — L’étudiant peut montrer son portfolio à n’importe qui (utilisateur Karuta ou non) en mode lecture seulement (aucun droit d’édition).

etudiant,all,?,4,unlimited,Partager@fr

Note 1 : Grâce au point d'interrogation (?), le destinataire n'est pas spécifié à l'avance ; lorsque l’étudiant clique sur Partager, une fenêtre pop-up lui permet de spécifier le courriel du destinataire.

Note 2 : L'instance de portfolio doit être partagée avec l'utilisateur public dans le rôle all.

Cas 2 — L’étudiant peut montrer son portfolio à n’importe qui (utilisateur Karuta ou non) en mode lecture seulement (aucun droit d’édition) mais le partage a une durée limitée de 24 heures.

etudiant,all,?,4,24,Partager@fr

Note 1 : Grâce au point d'interrogation (?), le destinataire n'est pas spécifié à l'avance ; lorsque l’étudiant clique sur Partager, une fenêtre pop-up lui permet de spécifier le courriel du destinataire.

Note 2 : Lors de l’instanciation du portfolio, le rôle all doit être partagé avec l’utilisateur public.

Cas 3 — L’étudiant demande à son professeur de procéder à son évaluation ; le professeur possède un compte Karuta et le portfolio lui est partagé dans le rôle professeur.

etudiant,professeur,professeur,1,12,Demander une évaluation@fr

Le professeur qui est partagé au portfolio reçoit un courriel et clique sur l’hyperlien pour procéder à l’évaluation.

Cas 4 — L’étudiant demande à son responsable de stage de compléter son évaluation ; le responsable de stage n’a pas de compte Karuta.

etudiant,evaluateur,yves.lacoste@airfrance.com,3,12,Demander une évaluation@fr

Le responsable de stage reçoit un courriel et clique sur l’hyperlien pour procéder à l’évaluation. Il n’a pas à s’identifier dans le système Karuta. L’hyperlien est valide pendant 12 heures.

Cas 5 — L’étudiant demande à un de ses collègues d’inscrire un commentaire dans son portfolio (évaluation par les pairs) ; le collègue possède un compte Karuta.

etudiant,pair,?,2,12,Demander un commentaire@fr

Le collègue reçoit un courriel et clique sur l’hyperlien pour écrire son commentaire.

Note : Grâce au point d'interrogation (?), le destinataire n'est pas spécifié à l'avance ; lorsque l’étudiant clique sur Partager, une fenêtre pop-up lui permet de spécifier le courriel du destinataire. On fait ici l’hypothèse que l’étudiant a un grand nombre de collègues et que le designer n’a pas spécifié à l'avance les collègues qui seront mobilisés, de façon à donner plus de latitude à l’étudiant dans le choix du pair qui l’évaluera.

Cas généraux : les deux tableaux ci-dessous expliquent les valeurs des paramètres de la métadonnée Partager.

Niveau Conditions
0 – utilisateur Karuta Le destinataire du courriel possède un compte Karuta et le portfolio lui est déjà partagé, dans le bon rôle. Son mot de passe lui sera demandé si il n'est pas déjà logué dans Karuta.
1 Le destinataire du courriel possède un compte Karuta et le portfolio lui est déjà partagé, dans le bon rôle. Son mot de passe ne lui sera pas demandé (si l'utilisateur n'est pas déjà logué dans Karuta, un login temporaire lui sera attribué).
2 Le destinataire du courriel possède un compte Karuta mais le portfolio ne lui est pas partagé. Le système partage le portfolio avec l’utilisateur dans le bon rôle. (Si l'utilisateur n'est pas déjà logué dans Karuta, un login temporaire lui sera attribué.)
3 Si le destinataire du courriel n'a pas de compte dans Karuta, un compte lui sera créé. Le système partage le portfolio avec l'utilisateur dans le bon rôle. (Si l'utilisateur n'est pas déjà logué dans Karuta, un login temporaire lui sera attribué.)
4 - lien public Si le destinataire du courriel n'a pas de compte dans Karuta, un compte lui sera créé. Le système partage le portfolio avec l'utilisateur dans le bon rôle. (Si l'utilisateur n'est pas déjà logué dans Karuta, un login temporaire lui sera attribué.) Lors de l’instanciation, le rôle all doit être partagé avec l’utilisateur public.

The metadata of an element allow the designer to give it a unique identifier and to define it's behavior.

The unique identifier is called the Semantic Tag and is used to reference an element from another element. The designer should attribute a unique identifier to every element added to a portfolio.

The next step is to specify the behavior of the element. For example, in the case of structural elements (Section, Page, Subsection), the designer may specify menus that allow the user to add elements to his or her portfolio. In the case of resources (the elements that contain the data), the designer can associate roles to each right, hence specifying who can do what on each element.

It is also in the metadata that the designer specifies which elements will populate a choice box, which users will be able to share their portfolio with another person (and under which security restrictions), create help bubbles, etc.

The metadata of an element is accessed through it's editing dialog box (pencil button).

Semantic Tag

An element's Semantic Tag is it's identifier and should express it's semantics (example: student-profile-page, student-name, etc). For each element the designer adds to a portfolio, he or she should replace the default semantic tag by one that expresses what the element's purpose is and what it will contain.

The semantic tag is used to search for an element in a portfolio. It is used in the Search metadata (see below) as well as in the report and batch scripts.

IMPORTANT: Search of a semantic tag is inclusive — for example, the search for all elements with semantic tag competency will also yield all elements with the semantic tag sub-competency, as the chain of characters of the first is included in the second.

Code

Most often attributed to Item and Subsection elements that will appear in single choice boxes (GetResource and GetGetResource elements) and multiple choice boxes (Get_Multiple and Import_Multiple functions). It is by the code that the user's selection is identified.

IMPORTANT: The code of a portfolio element must be unique within this portfolio.

By default, both the element's code and label are displayed in the choice box, and then only the label of the user's selection is displayed in the GetResource (or GetGetResource) element. This behavior may be modified by adding one or more of the following symbols to the element's code:

  • @ the code will not be displayed in the choice box
  • # the code will be displayed in the GetResource element
  • $ the label will not be displayed in the choice box
  • % the label will not be displayed in the GetResource element
  • ? the value will be displayed in the choice box (only Item elements have a field to indicate a value)
  • & the value will be displayed in the GetResource element (only Item elements have a field to indicate a value)
  • ! the element will not be selectable in the choice box (useful to create section titles in the choice box)

Multilingual Node

Specifies whether an element's label has a different value in each language (if the portfolio is multilingual). If Multilingual Node is not checked, there will only be one value for all languages, and that value will be stored in the default language (the default language is defined in the _init.js configuration file).

IMPORTANT: The code is never multilingual. It has the same value independently of the selected working language.

Multilingual Resource

Resource elements only. Specifies whether the data in the resource element has a different value in each language (if the portfolio is multilingual). If Multilingual Resource is not checked, there will only be one value for all languages, and that value will be stored in the default language (the default language is defined in the _init.js configuration file).

For example, a name or a birth date are usually not language dependant, whereas a text that gives instructions is definitely language specific.

Encrypt Resource

Resource elements only. Specifies whether the data in the resource element must be encrypted. If so, a key phrase is requested from the user when they open the portfolio. This phrase is used to encrypt/decrypt the data. It is not stored in the portfolio and thus must be memorized by the user, as he or she will be asked to provide it every time they open the portfolio.

See

List of roles, separated by a space character, that have the right to see the element. For example: student teacher.

By default, the right to see is attributed to the role all, which includes all roles defined on the portfolio. This is the only right to have a default value.

Delete

List of roles, separated by a space character, that have the right to delete the element. Displays the X button.

Edit Resource

Resource elements only. List of roles, separated by a space character, that have the right to edit the element (store and modify data in the element). Displays the pencil button.

Resource Without Pencil

Resource elements only. Does not display the pencil button to the roles that have the Edit Resource right. Used in combination with the Import_Multiple and Get_Multiple functions (see MENU FONCTIONS for details).

Comment

List of roles, separated by a space character, that have the right to comment the element. Adds a text editor at the bottom of the element's editing box. In the case of resource elements, the comment is timestamped and signed with the user's name. The user must also have the Edit Resource right on the element.

TIP: The designer may give him or herself the Comment right on a structural element — this way he or she may add the equivalent of a subtitle to the element's label.

Submit

List of roles, separated by a space character, that have the right to submit the element. Upon submission the Edit Resource and Delete rights are removed from the element and all it's children. Displays the Submit button.

Edit Code

List of roles, separated by a space character, that have the right to modify the code of the element.

Edit Label

List of roles, separated by a space character, that have the right to modify the label of the element.

Node Without Pencil

Does not display the pencil button to the roles that have the Edit Label right. Used in combination with the Import_Multiple and Get_Multiple functions (see MENU FONCTIONS for details).

Duplicate

List of roles, separated by a space character, that have the right to duplicate the element. The new element will be added at the end of the list of elements of the same hierarchical level in the tree. Displays the copy button.

Resize Image

Image elements only. List of roles, separated by a space character, that have the right to modify the size in which the image is displayed. Displays the pencil button.

Edit Proxy Target

Proxy elements only. List of roles, separated by a space character, that have the right to modify the element displayed by Proxy. The value of the original element is automatically updated.

Ordering

List of roles, separated by a space character, that have the right to modify the position of the element amongst the elements of the same hierarchical level in the tree. Displays the upward arrow button.

Move

List of roles, separated by a space character, that have the right to move the element to another brach in the portfolio tree. Displays the intersecting arrows button.

Move In

Semantic Tag that restricts the list of elements in which the element can be moved. Used in combination with the Move right.

yy

List of roles, separated by a space character, that have the right to zzz the element.

yy

List of roles, separated by a space character, that have the right to zzz the element.

Query

The Query field is used for:

  • choice boxes (Get_Resource and Get_Get_Resource elements)
  • custom menus on Sections, Pages and Subsections
  • choice boxes allowing multiple user choices implemented through menus (Get_Multiple, Get_Get_Multiple and Import_Multiple menus)

Format for Get_ (either Get_Resource or Get_Multiple)

portfolio_code.semantic_tag.label

  • portfolio_code: Code of rubrics portfolio in which to search for the specified semantic tag. Format is 'project code.rubrics portfolio code'. If 'project code.' is not present, search for the rubrics portfolio will be in the current project (project containing the Get_).
  • semantic_tag: Semantic tag of elements to search for in the rubrics portfolio.
  • label: Reserved word.

Format Get_Get_ (either Get_Get_Resource or Get_Get_Multiple)

portfolio_code.position.semantic_tag1.semantic_tag2.label

  • portfolio_code: Code of rubrics portfolio in which to search.
  • position: Position, in the current portfolio, of the parent element which will restrict the search in the rubrics portfolio. One of the following values: child, sibling, parent, parent.parent, parent.parent.parent
  • semantic_tag1: Semantic tag of parent element (restricts the search inside this parent element in the rubrics portfolio).
  • semantic_tag2: Semantic tag of elements to search for in the rubrics portfolio.
  • label: Reserved word.

Menu - Get_Multiple

This function allows the user to select multiple items from a list. For each item selected by the user, an reusable component is added to the portfolio. This component is usualy a Get_Resource element, but it may also be a structural element containing, amongst other things, the Get_Resource element. In any case, the user's choice is set by Karuta in the Get_Resource element.

Format

function,get_multiple/query/portfolio_code.semantic_tag /get_resource_semantic_tag,label@en,roles

  • function: Reserved word.
  • get_multiple: Reserved word.
  • query: Query of the Get_Resource element (see Query above).
  • portfolio_code.semantic_tag: Reusable component to add to the portfolio (Get-Resource element or more complex component containing the Get_Resource element).
  • get_resource_semantic_tag: Semantic tag of the Get_Resource element.
  • label@en: User menu label.
  • roles: List of roles that have this menu (separated by a space character).

Menu - Get_Get_Multiple

Same as the previous function, except the choices available to the user in the list are restricted by the selection in a parent element (as in Get_Get_Resource elements).

Format

function,get_get_multiple/query/ portfolio_code.semantic_tag/get_get_resource_semantic_tag, label@en,roles

  • function: Reserved word.
  • get_get_multiple: Reserved word.
  • query : Query of the Get_Get_Resource element.
  • portfolio_code.semantic_tag: Reusable component to add to the portfolio (Get_Get_Resource element or more complex component containing the Get_Get_Resource element).
  • get_get_resource_semantic_tag: Semantic tag of the Get_Get_Resource element.
  • label@en: User menu label.
  • roles: List of roles that have this menu (separated by a space character).
